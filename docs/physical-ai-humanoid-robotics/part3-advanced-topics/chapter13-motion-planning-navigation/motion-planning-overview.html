Chapter 13: Motion Planning and Navigation

Introductionâ€‹

Navigation is the science of getting from one place to another. For a robot, it's the process of answering three continuous questions:

  1. "Where am I?" (Localization)
  2. "Where am I going?" (The Goal)
  3. "How do I get there?" (Path Planning)

Motion Planning is the core of that third question. It is the process of computing a collision-free and dynamically feasible path from a start configuration to a goal configuration. This is a fundamental capability that ties together perception, mapping, control, and kinematics. Whether moving the entire robot through a room or just moving its arm to pick up an object, the robot must first plan a valid motion.

13.1 The Navigation Stackâ€‹

Robot navigation is typically structured as a layered system, often called a "navigation stack."

  • The Map: The foundation of navigation is a map of the environment, usually generated by a SLAM algorithm (Chapter 6). A common representation is a costmap, which is a 2D grid where each cell contains a value representing the "cost" of traversing that cell. Cells containing obstacles have an infinitely high cost, cells near obstacles have a high cost, and open space has a low cost.
  • The Global Planner: This component is responsible for finding a high-level path from the robot's current location to a distant goal. It operates on the overall map and finds an optimal path, like a GPS giving you driving directions across a city. It doesn't worry about dynamic obstacles or the robot's kinematics.
  • The Local Planner: This component is responsible for real-time collision avoidance. It takes the path from the global planner as a guide and computes the actual motor commands for the next few seconds. Its primary job is to generate dynamically feasible trajectories that avoid immediate obstacles (like a person walking in front of the robot), even if it means deviating temporarily from the global plan.

This layered approach allows the robot to make intelligent, long-range plans while remaining reactive to its immediate surroundings.

13.2 Global Path Planningâ€‹

The goal of the global planner is to find the "best" path on the costmap. This is a classic graph search problem.

  • Dijkstra's Algorithm: A famous algorithm that is guaranteed to find the shortest path from a starting point to all other points on a graph. It works by exploring outwards, always expanding the path to the nearest unvisited node.
  • A (A-star):* A* is the workhorse of global planning in robotics. It is an extension of Dijkstra's algorithm that is much more efficient. In addition to the actual cost to reach a node (like Dijkstra), A* uses a heuristicâ€”an educated guessâ€”to estimate the cost from that node to the final goal. This allows it to prioritize its search in the general direction of the goal, avoiding the need to explore in all directions.

13.3 Local Path Planning and Collision Avoidanceâ€‹

The local planner's job is to turn the global plan into action. A widely used algorithm for this is the Dynamic Window Approach (DWA). DWA works in a rapid, repetitive cycle:

  1. Sample Velocities: First, it creates a list of achievable linear and angular velocities that the robot can safely execute in the next short time step. This is the "dynamic window."
  2. Simulate Trajectories: For each pair of sampled velocities, it simulates the resulting curved trajectory for a short period into the future.
  3. Score Trajectories: Each simulated trajectory is evaluated and scored based on several criteria:
    • Obstacle Avoidance: How close does it get to an obstacle? (High penalty for getting too close).
    • Path Following: How well does it follow the global path?
    • Goal Progress: How much closer does it get to the local goal?
  4. Select and Execute: The trajectory with the best score is chosen, and the corresponding linear and angular velocity commands are sent to the robot's motor controllers.

This entire cycle repeats many times per second, allowing the robot to move smoothly while reacting to changes in its environment.

13.4 Motion Planning for Manipulationâ€‹

Planning for a multi-jointed arm is much more complex than planning for a mobile base. The challenge is the high dimensionality of the problem.

  • Configuration Space (C-Space): Imagine a 7-DOF robot arm. Its configuration can be described by a list of its 7 joint angles. The space of all possible configurations is a 7-dimensional space called the C-Space. An obstacle in the 3D real world (workspace) corresponds to a complex, high-dimensional obstacle region in C-Space. The motion planning problem is now transformed into finding a path for a single point from a start configuration to a goal configuration, while avoiding the C-Space obstacles.

Sampling-Based Plannersâ€‹

Because C-Space is too complex to represent explicitly, modern planners use sampling-based techniques.

  • Rapidly-exploring Random Tree (RRT): This is one of the most popular and effective algorithms for high-dimensional motion planning. RRT works by "growing" a tree of valid, collision-free configurations. It starts the tree at the arm's initial configuration. Then, it repeatedly samples a random configuration in the C-Space and tries to expand the nearest node in the tree towards that random sample. This causes the tree to rapidly explore the reachable space. When a path from the start to the goal is found, the plan is complete. Variants like RRT* continue to search for better and shorter paths after the first one is found.

13.5 Whole-Body Planningâ€‹

For a humanoid, we cannot plan for the arms and legs in isolation. Reaching for an object with an arm changes the robot's Center of Mass, which must be compensated for by the legs and torso to maintain balance. This requires Whole-Body Motion Planning.

This is the ultimate motion planning challenge, where the planner must find a single, dynamically stable, and collision-free trajectory for all 30+ joints of the robot simultaneously. It needs to decide not only where the arm should move, but also how the legs, torso, and head should move to support the action, maintain balance, and avoid collisions. This is an active area of advanced research that combines sampling-based planning with trajectory optimization and contact mechanics to generate the fluid, graceful, and purposeful movements that will allow humanoids to operate effectively in complex environments.
